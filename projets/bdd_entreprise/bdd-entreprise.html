Le projet

Concevoir une BDD de A à Z

Ce projet vise à concevoir une base de données pour une entreprise de distribution, englobant la gestion des stocks de produits, les commandes clients, les itinéraires de livraison, les types d'emballage, les véhicules et chauffeurs.

Le travail comprend l'analyse des besoins, la construction du schéma EA, la dérivation vers un modèle relationnel normalisé, la rédaction du dictionnaire de données, la création d'un jeu d'essai cohérent et la programmation de requêtes SQL complexes sous MySQL.

J'ai rencontré des difficultés dans la conception du schéma Entité-Association, notamment pour modéliser les relations N-N et les entités faibles, mais elles ont été surmontées grâce à des recherches approfondies. Travail réalisé seul.

Les 5 étapes de conception

🔍 Agrandir
01

Schéma Entité-Association

Modélisation de toutes les entités du système et de leurs relations : Produit, Fournisseur, Commande, Client, Type_Prod, Chauffeur, Tournée, Assurance, Contrat, Emballage, Type_Véhicule. Les relations N-N sont représentées avec leurs cardinalités et attributs portés.

📦 Produit 🚚 Chauffeur 📋 Commande 👥 Client 🏭 Fournisseur 🛣️ Tournée
Merise Entités / Relations Cardinalités Relations N-N
🔍 Agrandir
02

Graphe de dépendances fonctionnelles

Représentation des dépendances fonctionnelles entre tous les attributs du système. Ce graphe est l'étape clé qui permet d'identifier les clés primaires, de détecter les anomalies et de préparer la normalisation en 3ème forme normale (3NF).

Dépendances fonctionnelles Normalisation 3NF Clés primaires
🔍 Agrandir
03

Dictionnaire de données

Description précise de chaque attribut : nom, type de données, contraintes, description et entité d'appartenance. Ce document garantit la cohérence et la compréhension de la base par toute l'équipe technique. Note : non complet en raison de la contrainte de place.

Documentation Types de données Contraintes
🔍 Agrandir
04

Schéma Relationnel

Dérivation du schéma EA vers le modèle relationnel : chaque entité devient une table, les relations N-N génèrent des tables de jonction, les clés étrangères sont définies. La structure est prête pour l'implémentation sous MySQL.

Tables Clés étrangères Tables de jonction MySQL
🔍 Agrandir
05

Requêtes SQL complexes

5 requêtes métier démontrant la maîtrise du SQL avancé :

R1Quantité livrée de "Tournevis" en juin 2023 par commande
R2Kilométrage moyen des véhicules entre deux dates
R3Historique des prix de vente de "Tournevis"
R4Pourcentage de clients fidèles en 2022
R5Chauffeurs non-chauffeurs entre deux dates
JOIN GROUP BY SUM / AVG / COUNT WHERE / AND Sous-requêtes
